Panduan Setup (Zero to Hero)

Langkah teknis mendetail yang harus dilakukan di awal proyek. Lakukan secara berurutan.

1. Persiapan PC Runner (Mesin Lokal)

Siapkan PC khusus di studio. Jangan pakai PC yang sedang digunakan *artist/programmer* aktif.

  1. Install Unreal Engine (Versi harus sama persis dengan project).
  2. Install Visual Studio 2022 (Pilih workload "Game development with C++" saat instalasi).
  3. Install Git for Windows.
  4. Buka Git Bash / PowerShell, pastikan akun GitHub sudah *login* (via SSH/Credential Manager) agar PC bisa `git pull` tanpa minta password terus.

2. Daftarkan PC ke GitHub (Self-Hosted Runner)

Menghubungkan GitHub dengan PC lokal agar GitHub bisa memberi perintah "Mulai Build".

  1. Buka Repository GitHub > Settings > Actions > Runners (di menu kiri bawah).
  2. Klik tombol hijau New self-hosted runner. Pilih OS Windows.
  3. Buka PowerShell di PC Runner. Copy-paste block kode yang diberikan GitHub ke PowerShell satu per satu.
  4. Saat Configure minta nama, ketik misal: Studio-Build-PC.
  5. PENTING: Saat ditanya "run as service", ketik Y. (Agar runner nyala otomatis tiap PC dihidupkan).

3. Pasang Discord Webhook

Agar tim dapat notifikasi otomatis di Discord jika build selesai.

  1. Di Discord Server, klik Edit Channel > Integrations > Webhooks > New Webhook. Copy URL-nya.
  2. Kembali ke GitHub Repo > Settings > Secrets and variables > Actions.
  3. Klik New repository secret.
  4. Isi Name dengan DISCORD_WEBHOOK, lalu Paste URL Discord di kolom Secret. Save.

Git Branching Standard

Standar penamaan dan peruntukan cabang (branch) repository untuk menjaga integritas kode dan mencegah bentrok CI/CD.

Production

main / master

Cabang paling krusial. Berisi kode yang 100% stabil dan siap rilis ke publik.

  • Status: Protected Branch.
  • Larangan: Dilarang commit langsung ke cabang ini!
  • Aksi: Hanya menerima Pull Request (PR) dari staging atau hotfix.
  • CI/CD Trigger: Memicu Auto Production Build jika diberi Tag Release.
Integration

staging / develop

Pusat integrasi kode harian. Tempat berkumpulnya fitur dari berbagai programmer.

  • Status: Default Working Branch.
  • Aksi: Area bagi QA Tester untuk mencoba fitur yang baru digabungkan.
  • CI/CD Trigger: Target utama untuk dibuild menggunakan tombol Manual Staging Build.

feature/*

Cabang tempat programmer bekerja sehari-hari (contoh: feature/inventory).

  • Aturan: Selalu dibuat bercabang dari staging.
  • Akhir: Setelah fitur selesai, programmer membuat Pull Request untuk menggabungkannya kembali ke staging.

hotfix/*

Cabang darurat untuk memperbaiki bug kritis di versi rilis / production.

  • Aturan: Dibuat bercabang langsung dari main.
  • Akhir: Setelah fix selesai, Wajib di-merge kembali ke main DAN ke staging agar bug tidak terulang.

Pipeline Scripts (YAML)

Buat kedua file ini di PC kamu lalu push ke GitHub. Simpan di dalam folder .github/workflows/.

1. build_staging.yml

Jalur Manual (Menggunakan workflow_dispatch). Simpan hasil sebagai Artifact.

name: MANUAL BUILD - Staging/Dev
on:
  workflow_dispatch:
    inputs:
      branch_target:
        description: 'Branch to build (e.g., staging)'
        required: true
        default: 'staging'

env:
  FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: true

jobs:
  build_staging:
    runs-on: self-hosted
    steps:
      - name: Checkout Code
        uses: actions/checkout@v4
        with:
          ref: ${{ github.event.inputs.branch_target }}

      - name: Setup Variables
        run: |
          $DATE_STR = Get-Date -Format "yyyyMMdd-HHmm"
          $REPO_NAME = "${{ github.event.repository.name }}"
          echo "ZIP_NAME=${REPO_NAME}_Stg_${DATE_STR}.zip" >> $env:GITHUB_ENV
          echo "BUILD_DIR=${{ github.workspace }}\Build_Output" >> $env:GITHUB_ENV

      - name: UAT Build
        run: |
          $UAT_Path = "C:\Program Files\Epic Games\UE_5.3\Engine\Build\BatchFiles\RunUAT.bat"
          $ProjectFile = Get-ChildItem -Path "${{ github.workspace }}" -Filter *.uproject | Select-Object -First 1
          & $UAT_Path BuildCookRun -project="$($ProjectFile.FullName)" -noP4 -platform=Win64 -clientconfig=Development -cook -allmaps -build -stage -pak -archive -archivedirectory="${{ env.BUILD_DIR }}"

      - name: Compress
        run: Compress-Archive -Path "${{ env.BUILD_DIR }}\Windows\*" -DestinationPath "${{ env.ZIP_NAME }}" -Force

      - name: Upload Artifact
        uses: actions/upload-artifact@v4
        with:
          name: ${{ env.ZIP_NAME }}
          path: ${{ env.ZIP_NAME }}
          retention-days: 7

      - name: Notify Discord
        if: always()
        run: |
          $payload = @{
            content = "**🛠️ STAGING BUILD Selesai!**`n**Project:** ${{ github.event.repository.name }}`n**Branch:** `${{ github.event.inputs.branch_target }}` `n**Status:** ${{ job.status }}`n*Silakan cek tab Actions di GitHub untuk mendownload Artifact-nya.*"
          }
          Invoke-RestMethod -Uri "${{ secrets.DISCORD_WEBHOOK }}" -Method Post -Body ($payload | ConvertTo-Json) -ContentType "application/json"

2. build_production.yml

Jalur Otomatis (Terpicu saat buat Release). Nempel zip ke halaman rilis.

name: AUTO BUILD - Official Release
on:
  release:
    types: [published]

env:
  FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: true

jobs:
  build_prod:
    runs-on: self-hosted
    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Variables
        run: |
          $TAG_NAME = "${{ github.event.release.tag_name }}"
          $REPO_NAME = "${{ github.event.repository.name }}"
          echo "ZIP_NAME=${REPO_NAME}_${TAG_NAME}.zip" >> $env:GITHUB_ENV
          echo "BUILD_DIR=${{ github.workspace }}\Build_Output" >> $env:GITHUB_ENV

      - name: UAT Build
        run: |
          $UAT_Path = "C:\Program Files\Epic Games\UE_5.3\Engine\Build\BatchFiles\RunUAT.bat"
          $ProjectFile = Get-ChildItem -Path "${{ github.workspace }}" -Filter *.uproject | Select-Object -First 1
          & $UAT_Path BuildCookRun -project="$($ProjectFile.FullName)" -noP4 -platform=Win64 -clientconfig=Shipping -cook -allmaps -build -stage -pak -archive -archivedirectory="${{ env.BUILD_DIR }}"

      - name: Compress
        run: Compress-Archive -Path "${{ env.BUILD_DIR }}\Windows\*" -DestinationPath "${{ env.ZIP_NAME }}" -Force

      - name: Upload to Release
        uses: svenstaro/upload-release-action@v2
        with:
          repo_token: ${{ secrets.GITHUB_TOKEN }}
          file: ${{ env.ZIP_NAME }}
          asset_name: ${{ env.ZIP_NAME }}
          tag: ${{ github.ref }}
          overwrite: true

      - name: Notify Discord
        if: always()
        run: |
          $payload = @{
            content = "**🚀 PRODUCTION RELEASE Selesai!**`n**Project:** ${{ github.event.repository.name }}`n**Version:** `${{ github.event.release.tag_name }}` `n**Status:** ${{ job.status }}`n**Download Link:** ${{ github.event.release.html_url }}"
          }
          Invoke-RestMethod -Uri "${{ secrets.DISCORD_WEBHOOK }}" -Method Post -Body ($payload | ConvertTo-Json) -ContentType "application/json"

How To Execute (Cara Pakai)

Panduan eksekusi harian setelah semua setup selesai.

A

Cara Build Staging (Manual)

  1. Buka repository GitHub, klik tab Actions (di deretan atas).
  2. Di sidebar sebelah kiri, klik tulisan MANUAL BUILD - Staging/Dev.
  3. Di area kanan, klik tombol abu-abu bertuliskan Run workflow ▼.
  4. Ketik nama branch yang ingin di-build (misal ketik: staging).
  5. Klik tombol hijau Run workflow.
  6. Cara Download: Setelah selesai, klik riwayat run tersebut, scroll ke paling bawah halaman. Ada file zip di bagian Artifacts.
B

Cara Build Production (Release)

  1. Pastikan fitur sudah selesai dan dimerge ke branch main.
  2. Di halaman utama repository, lihat sidebar kanan, klik Releases.
  3. Klik tombol Draft a new release.
  4. Buat Tag baru (misal: v1.2.0).
  5. Tulis judul dan Release notes.
  6. Klik Publish release. (Begitu diklik, build otomatis berjalan!)
  7. Cara Download: Link download akan terlampir otomatis di halaman Release tersebut beserta link di Discord.
