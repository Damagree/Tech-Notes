Phase 1: Preparation & Setup

This section outlines the foundational one-time setup required to enable the automation pipeline. It details hardware prerequisites, GitHub configuration, and repository hygiene. Understanding this is crucial for DevOps or anyone maintaining the build infrastructure.

1. PC Build Preparation

Prepare a dedicated Windows PC/Laptop to execute the builds. This acts as the Self-Hosted Runner.

  • Unreal Engine installed (matching project version).
  • Visual Studio with "Game development with C++" workload.
  • Git for Windows installed.
  • Git Authentication: PC must pull from repo automatically via SSH key or Git Credential Manager.

2. Setup GitHub Runner

  1. Navigate to Settings > Actions > Runners in your GitHub Repo.
  2. Click New self-hosted runner.
  3. Select Windows (OS) and x64 (Architecture).
  4. Open PowerShell on the Build PC and run the provided commands (Download, Configure, Run).
  5. Recommendation: Install runner as a Windows Service for 24/7 availability.

3. .gitignore Configuration

Ensure the repository is not cluttered with temporary cache files. Place this in the root directory.

Phase 2: Versioning SOP

This section provides the interactive guidelines for Semantic Versioning (SemVer) and release tagging. Proper versioning is the trigger mechanism for the entire automation pipeline. The visual data below helps teams correlate tags with expected stability and target audiences.

Core Versioning Numbers

Format Name When to Increment
v1.x.x Major Breaking changes, architectural overhauls, core upgrades.
x.2.x Minor Adding new functionality (backwards-compatible).
x.x.3 Patch Hotfixes or small optimizations.

Release Trigger Process

  1. Merge: Ensure features are merged into target branch.
  2. Draft: Go to Repo > Releases > Draft a new release.
  3. Tag: Create tag (e.g., v0.2.0-beta).
  4. Target: Select branch (e.g., develop).
  5. Notes: Click Generate release notes.
  6. Publish: Click Publish release (This triggers CI/CD).

Phase 3 & 4: CI/CD & Discord

This section details the core automation script and notification integration. It provides the exact YAML configuration needed to command the self-hosted runner and ping the team on Discord upon completion.

💬 Discord Setup

  1. Open Discord Server Settings > Integrations.
  2. Click Webhooks > New Webhook ("Build Bot").
  3. Copy the Webhook URL.
  4. In GitHub Repo go to Settings > Secrets and variables > Actions.
  5. New repository secret:
    Name: DISCORD_WEBHOOK
    Value: [Paste URL]

Phase 4: Verification

  • Verify UAT_Path in YAML matches engine version.
  • Publish release with tag v0.1.0-alpha.
  • Watch Actions tab for progress.
  • Wait for Discord ping.

GitHub Actions YAML

Create at: .github/workflows/unreal_build.yml

name: Unreal Engine Build Automation

on:
  release:
    types: [published]

jobs:
  build_unreal:
    runs-on: self-hosted

    steps:
    - name: Checkout Code
      uses: actions/checkout@v4

    - name: Setup Variables
      id: vars
      run: |
        $TAG_NAME = "${{ github.event.release.tag_name }}"
        $REPO_NAME = "${{ github.event.repository.name }}"
        echo "ZIP_NAME=${REPO_NAME}_${TAG_NAME}.zip" >> $env:GITHUB_ENV
        echo "BUILD_DIR=${{ github.workspace }}\Build_Output" >> $env:GITHUB_ENV

    - name: Run Unreal Automation Tool
      run: |
        $UAT_Path = "C:\Program Files\Epic Games\UE_5.x\Engine\Build\BatchFiles\RunUAT.bat"
        $ProjectFile = Get-ChildItem -Path "${{ github.workspace }}" -Filter *.uproject | Select-Object -First 1
        if (-not $ProjectFile) { exit 1 }
        & $UAT_Path BuildCookRun -project="$($ProjectFile.FullName)" -noP4 -platform=Win64 -clientconfig=Development -cook -allmaps -build -stage -pak -archive -archivedirectory="${{ env.BUILD_DIR }}"

    - name: Compress Build Output
      run: |
        Compress-Archive -Path "${{ env.BUILD_DIR }}\Windows\*" -DestinationPath "${{ env.ZIP_NAME }}" -Force

    - name: Upload Build to Release
      uses: svenstaro/upload-release-action@v2
      with:
        repo_token: ${{ secrets.GITHUB_TOKEN }}
        file: ${{ env.ZIP_NAME }}
        asset_name: ${{ env.ZIP_NAME }}
        tag: ${{ github.ref }}
        overwrite: true

    - name: Discord Notification
      if: always()
      uses: appleboy/discord-action@master
      with:
        webhook_id: ${{ secrets.DISCORD_WEBHOOK }}
        message: |
          **🚀 New Build Available!**
          **Project:** ${{ github.event.repository.name }}
          **Version:** ${{ github.event.release.tag_name }}
          **Status:** ${{ job.status == 'success' && '✅ Success' || '❌ Failed' }}
          **Download:** ${{ github.event.release.html_url }}
          ---
          *Triggered by: ${{ github.actor }}*
